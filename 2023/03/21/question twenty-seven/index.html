2.最主要看的是class.php文件，class.php有三个类，分别是User，FileList，File，看到File中有close()中有一个file_get_contents()函数，可以打开文件，寻找可以使用close函数的地方，看到User类中有

这样就有一个逻辑：User=>__destruct()=>File=>close()，试了试发现这个逻辑并不能

3.在FileList类中有一个__call魔术方法

这里的__call魔术方法，如果要调用的方法我们这个类中不存在，就会去File中找这个方法，并把执行结果存入 $this->results[$file->name()][$func]里面

4.最终的链子是$user -> __destruct() => $db -> close() => $db->__call(close) => $file -> close() =>$results=file_get_contents($filename) => FileList->__destruct()输出$result

__destruct可以把$this->results[$file->name()][$func]的内容打印出来

让$db等于new FileList()，File中的filename等于/flag.txt

运行这个脚本会自动生成shell.phar文件，将shell.phar文件后缀改为jpg，然后上传，抓取点击删除时的包
